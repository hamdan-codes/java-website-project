Java-Class
Java is an object - oriented programmin language

A class is kind of mold that dictates what objects can and cannot do.
It is is a template or blueprint for how to build an object.

In simple words a class is prototype that defines state placeholders and behaviour which is common to all objects of its kind.

Everything in Java is associated with classes and objects, along with its attributes and methods. For example: in real life, a car is an object. The car has attributes, such as weight and color, and methods, such as drive and brake. An object is called as an instance of a class.
If one class is defined it is posiible to create many isnatnces of the class.




General format to declare a class:

public class < class_name >
{
data member;
method;
}
The class is a keyword and class_name can be any valid name.
The members of the class are data members and methods.

Data members represents the variables and method refers to the functionality.

Java Class Attributes
Class Attributes are variable defined/declared in the scope of a class.

Attributes are variables within a class.
An attribute is another term for a field.
It's typically a public constant or a public variable that can be accessed directly.
These attributes are allocated with memory only, when an object of a class is created.




Let's see an example about class attribute.
The following code segment contains the definition for a class named Student.

class Student
{
String name;
int roll_no;
public static void main(String[] args) {
Student s = new Student();
s.name = "Aakash";
s.roll_no = 1905147;
System.out.println(s.name);
System.out.println(s.roll_no);
}
}
In class Student the two attributes are name and roll_no.
Here, as no access specifier is given, it is considered to be of default type.
These attributes are accessed by object useng "." operator.
Only those data members are accessible by object which are public or default type.





Java-Member Function
Member Functions are functions declared/defined in the scope of a class.

Member functions are of following types:

  • Built-in - (Predefined in java)
  • User-defined - (Built by users, and can be used at any point of time)
  • Static functions - ( it can be accessed without creating an object of the class)
  • abstract functions - (declared in abstract class. it is used a template and data abstraction.)




Let's see an example for creating a function.
The following code segment contains the definition for a class named Student

class Student
{
String name;
int roll_no;
public void getdata() {
name = "Aakash";
roll_no = 1905147;
}
public void showdata(){
System.out.println("name: "+ name+" roll no: "+ roll_no);
}
public static void main(String[] args) {
Student s = new Student();
s.getdata();
s.showdata();
}
}
Here two functions getdata() and putdata() are declared.
getdata() is public and of void type. And its work is to initialise name and roll_no with some data.
putdata() is also public and of void type. And its work is to print the values of name and roll_no.
Here both functions are public type, which can be directly accessed by main method, using object and "." operator.





Java-Modifiers
It is used to set the access level for classes, attributes, methods and constructors.

For class the access specifiers are:

Modifiers Description
public The class is accessible by any other class
default The class is only accessible by classes in the same package.
This is used when you don't specify a modifier.


For attributes, methods and constructors the access specifiers are:

Modifiers Description
public The code is accessible for all classes
default The class is only accessible by classes in the same package.
This is used when you don't specify a modifier.
private The code is only accessible within the declared class
protected The code is accessible in the same package and subclasses.




Let's see an example.
The following code segment contains the definition for a class named Student

class Student
{
String name;
private roll_no;
public void getdata() {
name = "Aakash";
roll_no = 1905147;
}
public void showdata(){
System.out.println("name: "+ name+" roll no: "+ roll_no);
}
public static void main(String[] args) {
Student s = new Student();
s.getdata();
s.showdata();
}
}
Here Student class is default type as no access specifier is given to class.
In class Student we have two attributes name and roll_no.
As there is no access specifier provided name avriable, it has pre-defined access specifier, i.e. , default.
Whereas, attribute roll_no is private type which is only visible to the scope of class. And can't be accessed by main function.
The two functions getdata() and putdata() are predefined public type.
And therefore, it can be directly accessed by main function.
